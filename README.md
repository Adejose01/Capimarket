# CapiMercado - Open Source E-Commerce Ecosystem

CapiMercado is a premium, open-source web application designed to act as a multi-vendor marketplace platform. Built with a modern, high-performance tech stack, it provides an elegant, app-like experience (Mobile-First, "Figma Premium UX") and a scalable backend architecture.

## 🚀 Tech Stack

- **Frontend:** React 18 + Vite
- **Styling & UX:** Tailwind CSS 4 + Framer Motion (Spring Physics & Animations)
- **Routing:** React Router v6 (Modal-Route implementation)
- **Backend / Database:** PocketBase v0.23+ (Go + SQLite WAL)
- **Infrastructure:** Docker & Docker Compose (Nginx reverse proxy ready)

## ✨ Features

- **Full-Width Premium UI:** Smooth layout transitions, slide-up modal routing, and glassmorphism elements.
- **Dynamic Category & Store Filtering:** Instant client-side chained filtering (`AND` logic) for Categories, Prices, Location, and Condition.
- **Store Portals:** Fully customizable vendor pages with dynamic banners, logos, and custom hex-color themes.
- **Real-time Map Integration:** Vendors can attach Google Maps URLs that dynamically render in the UI.
- **WhatsApp Integration:** Built-in order checkout flows directly to the vendor's WhatsApp.
- **Skeleton Loading States:** Sleek, animated skeleton loaders for immediate perceived performance without visual jumps.
- **Admin & Super-Admin Tiers:** PocketBase permissions and GUI for full data and seller approval management.

## 🛠️ Quick Start (Development)

The repository is fully ready to be cloned and launched. **No manual database setup is required.** The database schema is bundled as automated PocketBase JS migrations.

### Prerequisites
- Docker & Docker Compose installed.

### 1. Clone the repository
```bash
git clone https://github.com/your-username/capimercado.git
cd capimercado
```

### 2. Launch the Application
This command will build the Vite frontend and the PocketBase backend, and automatically apply all database schemas via migrations:

```bash
docker-compose up -d --build
```

### 3. Access the Apps
- **Marketplace Frontend:** [http://localhost:3000](http://localhost:3000)
- **PocketBase Admin Panel:** [http://localhost:8090/_/](http://localhost:8090/_/)

> *Note: On the first boot, create your Super Admin account in the PocketBase dashboard. The schema will be fully populated with `users`, `stores`, `products`, and `categories` tables.*

## 🔒 Security & Data
The `pb/pb_data/` directory (where SQLite stores active data) is intentionally ignored in `.gitignore`. This ensures your production or test data is never committed to GitHub. Any user who clones this repository will start with a fresh database structure automatically generated by the `/pb_migrations/` files.
